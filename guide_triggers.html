Triggers

Come progettare trigger SQL da zero: sintassi, timing, granularità, e come risolvere gli esercizi tipici del Politecnico con esempi reali.

Cos'è un trigger?

Un trigger è una procedura che il database esegue automaticamente in risposta a un evento (INSERT, UPDATE, DELETE) su una tabella. È come un "riflesso condizionato" del database: quando succede X, fai Y.

Esempio pratico: ogni volta che viene inserito un nuovo ordine in un e-commerce, aggiorna automaticamente le scorte di magazzino. Senza trigger, dovresti farlo manualmente nel codice applicativo — con il rischio di dimenticartelo.

1. Sintassi completa di un trigger

CREATE TRIGGER nome_trigger
AFTER | BEFORE  -- quando si attiva
INSERT | UPDATE [OF colonna] | DELETE  -- quale evento
ON nome_tabella  -- su quale tabella
[REFERENCING NEW AS new_row OLD AS old_row]
[FOR EACH ROW | FOR EACH STATEMENT]  -- granularità
[WHEN (condizione)]  -- condizione opzionale
BEGIN
  -- corpo del trigger
END;

2. BEFORE vs AFTER

TimingQuando si attivaPuò modificare NEW?Uso tipico
BEFOREPrima che la modifica venga applicata al DBSÌValidazione, correzione valori, blocco operazione con RAISE EXCEPTION
AFTERDopo che la modifica è già nel DBNOPropagazione effetti, aggiornamento tabelle correlate, log
Regola pratica

Usa BEFORE se devi bloccare/modificare l'operazione. Usa AFTER se devi aggiornare altre tabelle in conseguenza. All'esame quasi sempre si usa AFTER.

3. FOR EACH ROW vs FOR EACH STATEMENT

GranularitàEseguitoAccesso a OLD/NEWUso tipico
FOR EACH ROWUna volta per ogni tupla modificataSÌ (OLD e NEW sono singole tuple)Quasi sempre — permette di sapere i valori della riga
FOR EACH STATEMENTUna volta per tutta l'istruzione SQLNO (solo tabelle OLD TABLE / NEW TABLE)Operazioni su insiemi, statistiche aggregate
All'esame del Poli

Quasi tutti gli esercizi richiedono FOR EACH ROW. È la granularità più comune e più flessibile. FOR EACH STATEMENT si usa raramente, di solito solo se esplicitamente richiesto.

4. OLD e NEW — Accesso ai valori della riga

All'interno di un trigger FOR EACH ROW puoi accedere ai valori della riga:

VariabileINSERTUPDATEDELETE
NEW.colonnaNuova riga inseritaValori dopo la modificaNon disponibile
OLD.colonnaNon disponibileValori prima della modificaRiga eliminata
Esempio

Tabella PERFORMANCE(MOVIEID, PID, ROLE, salary). Trigger su INSERT:
NEW.salary = il salary della riga appena inserita.
NEW.PID = il PID del performer appena aggiunto.

5. Quando creare un trigger — Matrice degli eventi

All'esame ti danno una tabella da mantenere (o una view materializzata) e devi capire su quali eventi e tabelle creare trigger. Ragiona così:

"Quale modifica a quale tabella può cambiare il contenuto della tabella target?"

Esempio — TDE Febbraio 2024

Vista da materializzare: SELECT P.PID, COALESCE(SUM(salary),0) AS cost FROM PROJECT P LEFT JOIN ASSIGNMENT A ON P.PID=A.PID LEFT JOIN EMP E ON E.EID=A.EID GROUP BY P.PID

TabellaINSERTUPDATEDELETE
PROJECTSÌ (nuovo progetto → riga in BUDGET con cost=0)NO (PK non cambia per assunzione)SÌ (progetto cancellato → rimuovi da BUDGET)
EMPNO (non ha assignment → non cambia BUDGET)SÌ (cambia salary → aggiorna BUDGET del progetto)NO (assignment viene eliminato prima)
ASSIGNMENTSÌ (nuovo assignment → aggiunge salary al progetto)NO (PK non cambia)SÌ (assignment rimosso → sottrae salary)

6. Metodo passo-passo per scrivere un trigger

Scegli l'evento: INSERT / UPDATE [OF col] / DELETE su quale tabella?

Usa la matrice degli eventi per capirlo.

Scegli il timing: BEFORE o AFTER?

Se devi bloccare l'operazione → BEFORE. Se devi propagare effetti → AFTER.

Scegli la granularità: FOR EACH ROW (quasi sempre)
Scrivi il corpo: cosa deve fare il trigger?

Usa NEW.col e OLD.col per accedere ai valori. Puoi fare SELECT, INSERT, UPDATE, DELETE su altre tabelle. Se devi bloccare → usa RAISE EXCEPTION.

Verifica la terminazione

Il tuo trigger attiva altri trigger? Costruisci il grafo di attivazione e verifica che sia aciclico (o che i cicli si interrompano).

7. Terminazione di un sistema di trigger

Un sistema di trigger termina se, per qualsiasi sequenza di operazioni, l'esecuzione dei trigger finisce in un numero finito di passi. Se un trigger attiva un altro trigger che a sua volta attiva il primo, si ha un loop infinito.

Grafo di attivazione

Disegna un grafo dove:

  • Ogni nodo è un trigger
  • C'è un arco da T1 a T2 se l'azione di T1 può attivare T2

Se il grafo è aciclico → terminazione garantita. Se ci sono cicli → analizza se la condizione WHEN li interrompe prima o poi (condizione sufficiente ma non necessaria per terminare).

Esempio — TDE 2023

T_total_salary (AFTER INSERT ON PERFORMANCE) → fa UPDATE su PERSON
T_richest (AFTER UPDATE OF total_salary ON PERSON) → fa UPDATE su RICHEST

Grafo: T_total_salary → T_richest → (nessun altro trigger). Aciclico → termina garantito.

8. Esempio completo — Mantenere la tabella BUDGET

Schema: PROJECT, EMP(salary), ASSIGNMENT. Da mantenere: BUDGET(PID, cost).

-- T1: nuovo progetto inserito → crea riga in BUDGET con cost=0
CREATE TRIGGER T1
AFTER INSERT ON PROJECT
FOR EACH ROW
BEGIN
  INSERT INTO BUDGET VALUES(NEW.PID, 0);
END;

-- T2: progetto eliminato → rimuovi da BUDGET
CREATE TRIGGER T2
AFTER DELETE ON PROJECT
FOR EACH ROW
BEGIN
  DELETE FROM BUDGET WHERE PID = OLD.PID;
END;

-- T3: salary di un dipendente aggiornato → aggiorna il BUDGET del progetto
CREATE TRIGGER T3
AFTER UPDATE OF SALARY ON EMP
FOR EACH ROW
BEGIN
  UPDATE BUDGET
  SET COST = COST + NEW.SALARY - OLD.SALARY
  WHERE PID = (SELECT PID FROM ASSIGNMENT WHERE EID = NEW.EID);
END;

-- T4: nuovo assignment → aggiungi salary al budget del progetto
CREATE TRIGGER T4
AFTER INSERT ON ASSIGNMENT
FOR EACH ROW
BEGIN
  UPDATE BUDGET
  SET COST = COST + (SELECT SALARY FROM EMP WHERE EID = NEW.EID)
  WHERE PID = NEW.PID;
END;

-- T5: assignment eliminato → sottrai salary dal budget
CREATE TRIGGER T5
AFTER DELETE ON ASSIGNMENT
FOR EACH ROW
BEGIN
  UPDATE BUDGET
  SET COST = COST - (SELECT SALARY FROM EMP WHERE EID = OLD.EID)
  WHERE PID = OLD.PID;
END;

9. Esempio — Aggiornare il performer più ricco (RICHEST)

Ogni volta che total_salary cambia in PERSON, aggiorna la tabella RICHEST con il PID del performer con salary massimo.

CREATE TRIGGER richest
AFTER UPDATE OF total_salary ON PERSON
FOR EACH ROW
DECLARE
  rich INT;
  tot  INT;
BEGIN
  -- recupera il PID correntemente nella tabella RICHEST e il suo salary
  SELECT R.PID, P.total_salary INTO rich, tot
  FROM RICHEST R JOIN PERSON P ON R.PID = P.PID;

  IF rich IS NULL THEN
    -- RICHEST è vuota: inserisci il primo
    INSERT INTO RICHEST VALUES (NEW.PID);
  ELSE
    IF tot < NEW.total_salary THEN
      -- il richest attuale è superato: aggiorna
      UPDATE RICHEST SET PID = NEW.PID;
    END IF;
  END IF;
END;

10. Esempio — Mantenere Top-5 MOST_VERSATILE

Ogni volta che un performer aggiunge un ruolo in un film, aggiorna la tabella MOST_VERSATILE con i 5 (PID, MOVIEID) con più ruoli nello stesso film. In caso di parità al 5° posto, includi tutti i pareggiati.

CREATE TRIGGER most_versatile
AFTER INSERT ON PERFORMANCE
FOR EACH ROW
DECLARE
  n_roles INT;
  fifth   INT;
BEGIN
  -- conta i ruoli del performer nel film appena aggiornato
  SELECT COUNT(*) INTO n_roles
  FROM PERFORMANCE
  WHERE PID = NEW.PID AND MOVIEID = NEW.MOVIEID;

  -- aggiorna o inserisce la riga (PID, MOVIEID) in MOST_VERSATILE
  IF EXISTS (SELECT 1 FROM MOST_VERSATILE
             WHERE PID = NEW.PID AND MOVIEID = NEW.MOVIEID) THEN
    UPDATE MOST_VERSATILE SET number_of_roles = n_roles
    WHERE PID = NEW.PID AND MOVIEID = NEW.MOVIEID;
  ELSE
    INSERT INTO MOST_VERSATILE VALUES (NEW.PID, NEW.MOVIEID, n_roles);
  END IF;

  -- trova il numero di ruoli del 5° classificato attuale
  SELECT MIN(number_of_roles) INTO fifth
  FROM (SELECT number_of_roles
        FROM MOST_VERSATILE
        ORDER BY number_of_roles DESC LIMIT 5) sub;

  -- rimuovi tutti quelli sotto il 5° (non in pareggio)
  DELETE FROM MOST_VERSATILE
  WHERE number_of_roles < fifth;
END;
Nota sul Top-5 con parità

L'esercizio richiede che in caso di parità al 5° posto vengano inclusi tutti i pareggiati. Per questo si elimina solo chi ha un numero di ruoli strettamente minore del minimo nei top-5, non chi è uguale al minimo.
